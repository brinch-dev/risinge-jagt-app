# Session 2026-05-13b — Live overvaagning, auto-checkin, event-graenser, bugfixes

## Hvad blev lavet

### Feature 1: Admin live-overvaagning
- Ny `user_locations` tabel i Supabase med realtime
- `LiveLocationService` sender bruger-positioner til Supabase hvert 15 sek
- Ny admin-side "Live overvaagning" med kort der viser alle brugeres positioner
- Blaa prikker = aktiv (inden for 5 min), graa = inaktiv
- Tap paa bruger viser navn, sidst set, position, nojagtighed
- RLS: brugere kan kun opdatere egen position, admin kan se alle

### Feature 2: GPS automatiseret, play-knap fjernet
- GPS tracking starter automatisk naar kortet aabnes
- Foreground service starter automatisk
- Position uploades til Supabase for live-overvaagning
- Ingen play/stop-knap mere

### Feature 3: Auto-checkin inden for 50m
- Naar bruger er inden for 50m af et event-omraade (paa eventdagen, tilmeldt)
- Groen "Check ind"-knap vises automatisk paa kortet
- Checkin gemmes i `event_checkins` tabel
- Logges i admin log som 'checkin' type
- Efter checkin vises bekraeftelsesbesked

### Feature 4: Event-graenser med polygon-tegning
- Admin kan tegne praecise jagtgraenser paa kort ved event-oprettelse
- Tegne-knap (blaa) aktiverer tegne-mode, tryk punkter paa kortet
- Fortryd (orange) og slet (graa) knapper
- Minimum 3 punkter for polygon
- Punkt-nummerering med groen start-punkt
- Graenser gemmes i `event_boundaries` tabel
- Vises som roede polygoner paa hovedkortet for kommende events

### Feature 5: Privat chat forenklet
- Tryk direkte paa en person → samtale oprettes med deres navn
- Ingen kaldenavn-felt for private chats
- Gruppe-chats har stadig navnefelt + checkboxes
- Egen bruger filtreres fra listen

### Bugfixes
1. **Cirkulaer RLS policy** — "Members can see co-members" paa channel_members foraarsagede infinite recursion. Fjernet og erstattet med simpel admin-policy.
2. **Live map refresh** — `ref.read(...notifier).build()` erstattet med `ref.invalidate()`
3. **Geofence log spam** — Admin logs blev skrevet hvert 5. sek. Nu throttled til max 1 pr. minut pr. omraade.
4. **Privat chat dobbelt-klik** — ListTile disabled under loading saa man ikke kan oprette flere kanaler
5. **Event reminder forkert ID** — `scheduleEventReminder` brugte event med tom id, nu bruger det oprettede event
6. **Exact alarm permission fjernet** — `requestExactAlarmsPermission()` aabnede en system-side brugeren ikke kunne navigere. Unodvendig da vi bruger `inexactAllowWhileIdle`.
7. **Broadcast trigger fejl** — `NEW.message` → `NEW.body`, `NEW.created_by` → `NEW.sender_id`, `'event'` → `'new_event'`
