# Session 2026-05-13 — Push Notifications & Chat RLS Fix

## Hvad blev lavet

### 1. Chat creation RLS fix
- **Problem**: Oprettelse af private/gruppe-chats fejlede med "row violates row-level security"
- **Aarsag**: SELECT-policyen paa `chat_channels` krævede at brugeren var i `channel_members`, men members blev foerst tilfojet EFTER `.select().single()` kaldet
- **Fix**: Tilfoejede `created_by = auth.uid()` til SELECT-policyen saa opretteren altid kan se sin egen kanal
- **Filer**: `supabase/schema.sql`, `supabase/fix_chat_rls.sql`

### 2. FCM Push Notification — server-side (Edge Function)
- **Oprettet Supabase Edge Function** `send-push` der sender FCM push via Firebase Cloud Messaging V1 API
- **Ingen eksterne dependencies** — bruger kun Deno built-ins og direkte REST-kald
- **Haandterer 3 typer**:
  - `chat_message` → push til alle kanal-medlemmer (undtagen afsender)
  - `broadcast` → push til alle brugere (undtagen afsender)
  - `event_notification` → push til alle brugere
- **Auto-oprydning**: Fjerner ugyldige FCM tokens (UNREGISTERED/NOT_FOUND)
- **Firebase auth**: JWT-baseret service account authentication (RS256)
- **Filer**: `supabase/functions/send-push/index.ts`

### 3. Database triggers for automatisk push
- Trigger paa `chat_messages` INSERT → kalder Edge Function med besked-data
- Trigger paa `app_notifications` INSERT → kalder Edge Function for broadcasts/events
- Bruger `pg_net` extension til asynkrone HTTP-kald
- **Filer**: `supabase/push_triggers.sql`

### 4. Deployment
- Supabase CLI linket til projekt `zbmpptfddowmchuyrrea`
- Firebase service account gemt som Supabase secret `FIREBASE_SERVICE_ACCOUNT`
- Edge Function deployed med `--no-verify-jwt` (kaldes fra database triggers)
- **Status**: Deployed og klar til test

## Hvor vi starter naeste gang

### Test push-notifikationer
1. Log ind paa 2 telefoner med forskellige brugere
2. Begge brugere skal have FCM token i `fcm_tokens` tabel
3. Send chat-besked fra bruger A → bruger B skal modtage push
4. Send broadcast fra admin → alle skal modtage push
5. Tjek Edge Function logs: `supabase functions logs send-push`

### Hvis push ikke virker — fejlsoegning
- Tjek at `fcm_tokens` har tokens for begge brugere
- Tjek Edge Function logs for fejl
- Tjek at Firebase service account secret er korrekt sat
- Tjek `pg_trigger` tabellen for aktive triggers

### Test-bruger setup
- Opret test-bruger i Supabase Auth med "Auto Confirm User" checked
- Saet rolle til `member` i `profiles` tabel
- Login paa 2. telefon med test-bruger
